# Version 5

**Start Date:** March 10, 2026
**End Date:** Ongoing

---

# Overview

Version 5 introduces the beginning of a **tool ecosystem** inside the application.

This update adds a new **Tool Section** that will contain various tools to enhance the overall functionality of the application. These tools are designed to support developer workflows and productivity without requiring external applications.

The first tool introduced in this system is the **API Tool Mode**, a lightweight API testing utility inspired by Postman but designed to be simpler and faster for quick API interactions.

This version lays the groundwork for integrating more tools in future updates.

---

# New Features

### Tool Section

A new **Tool Section** button has been added to the application interface.

This section acts as a central place where additional tools will be integrated. It serves as a foundation for expanding the application beyond its core functionality and allows new tools to be added without affecting the main workflow.

The Tool Section will continue to grow as more developer utilities and productivity tools are implemented in future versions.

---

### API Tool Mode

The first tool added to the Tool Section is **API Tool Mode**.

This feature provides a lightweight environment for interacting with APIs. It allows users to manage API projects and work with endpoints in a simplified interface inspired by Postman.

The goal of this tool is to provide a faster and more minimal alternative for basic API testing directly inside the application.

---

### API Project Creation

Users can now create API entries inside the tool. Each API can be registered with a name and a base URL, allowing endpoints to be organized under a specific API project.

This structure helps users manage multiple APIs and keep their endpoints organized within the tool.

---

### Swagger Import

API Tool Mode includes the ability to **import Swagger/OpenAPI specifications**.

When a Swagger source is provided, the tool automatically reads the API definition and retrieves all available endpoints. These endpoints are then organized within the API project, allowing users to easily view and interact with them.

This feature simplifies API exploration by automatically generating endpoint lists based on the Swagger specification.

---

# Notes

Version 5 marks the start of expanding the application into a **multi-tool development platform**.

The introduction of the **Tool Section** enables future integrations, while the **API Tool Mode** provides the first developer-focused utility within the application.

Future updates will continue expanding this system with additional tools and improvements.
