# Overview

Version 2 introduces major improvements to file searching, filtering, and usability.
This update focuses on **better control over scanned files, improved UI experience, and enhanced workflow efficiency**.

New features such as **Dark/Light mode**, **folder filtering**, and **improved file extension detection** make the tool more flexible when working with large or structured projects.

---

# Improvements

### Improved Search Accuracy

Search results are now clearer and more precise, helping users quickly locate relevant files.

### Better File Matching

Matching logic has been refined to detect files more accurately based on selected filters.

### Directory-Based Searching

The system now supports searching within specific directories such as:

`Backend/src/models`

This allows more structured and targeted scanning instead of relying only on file names.

### Improved File Extension Detection

File type detection has been improved to better recognize extensions and avoid incorrect matches.

### Enhanced User Interface

The UI has been refined for better readability, cleaner layout, and smoother interaction.

---

# New Features

### Dark Mode & Light Mode

Users can now switch between **Dark Mode** and **Light Mode** for a more comfortable viewing experience.

---

### File Extension Filter

Select specific file extensions to include in searches.

Examples:

* `.js`
* `.ts`
* `.cs`
* `.py`

This helps narrow results to only relevant file types.

---

### Ignore Extension Filter

Temporarily exclude specific file extensions from search results.

Useful when scanning large projects where certain file types are not needed.

---

### Folder Ignore Mode

Users can now **exclude specific folders** from being scanned.

Examples:

* `node_modules`
* `.git`
* `dist`
* `build`

This significantly improves performance when scanning large repositories.

---

### Folder Focus Mode

Users can select **specific folders to prioritize or scan exclusively**.

Example:

`Backend/src/models`

This allows precise searches within specific project modules.

---

### Refresh Folder Structure

A **Refresh** function allows the application to reload the current folder structure.

This is useful when:

* Files are added
* Files are deleted
* Files are modified

No application restart is required.
